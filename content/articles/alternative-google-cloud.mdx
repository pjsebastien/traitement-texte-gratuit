Google Cloud Platform (GCP) est un acteur majeur du cloud computing. Mais il n'est pas adapté à tous les besoins. Tarification complexe, données hébergées aux États-Unis, dépendance envers Google. Plusieurs alternatives méritent qu'on s'y attarde.

<Callout type="tip" title="Le résumé en 10 secondes">
**AWS** domine le marché avec le plus large catalogue de services. **Microsoft Azure** s'intègre parfaitement avec l'environnement Microsoft. **Hetzner** propose des serveurs européens à prix cassés. **OVHcloud** est le leader français du cloud souverain. **DigitalOcean** mise sur la simplicité. **Scaleway** est le cloud européen pensé pour les développeurs.
</Callout>

## Pourquoi chercher une alternative à Google Cloud ?

GCP propose plus de 200 services. C'est à la fois sa force et son problème. La facturation est opaque, les coûts peuvent exploser sans prévenir. Voici les raisons principales de regarder ailleurs :

- **Tarification imprévisible** : le modèle à l'usage rend les factures difficiles à anticiper. Des surprises à quatre ou cinq chiffres, ça arrive
- **Complexité** : la console GCP est dense. Configurer un simple serveur demande de naviguer dans des dizaines de menus
- **Données aux États-Unis** : malgré les régions européennes, Google reste soumis au Cloud Act américain
- **Fermetures de services** : Google a un historique chargé en matière de services abandonnés. Pas rassurant quand on bâtit une infrastructure dessus
- **Support** : le support gratuit est quasi inexistant. Le support payant démarre à 29 dollars par mois

## AWS : le plus grand catalogue de services

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="AWS, leader mondial du cloud computing" />

<AffiliateLink href="https://aws.amazon.com/fr/">Amazon Web Services (AWS)</AffiliateLink> est le leader mondial du cloud avec environ 31% de parts de marché. Le catalogue comprend plus de 200 services : calcul, stockage, bases de données, machine learning, IoT, blockchain.

L'offre gratuite (Free Tier) donne accès à une instance EC2 t2.micro pendant 12 mois, 5 Go de stockage S3, et une base de données RDS. De quoi tester sans sortir la carte bleue.

AWS est présent dans 33 régions géographiques, dont Paris (eu-west-3) et Francfort. La documentation est massive. La communauté aussi.

Le problème : la facturation est encore plus complexe que chez Google. Et le lock-in est réel. Une fois vos services configurés sur AWS, migrer ailleurs demande un effort considérable.

<Callout type="success" title="Ce qui le rend unique">
Plus grand catalogue de services cloud au monde. 33 régions géographiques. Free Tier généreux (12 mois). Communauté et documentation gigantesques. Leader du marché, donc très mature. Services d'IA et de machine learning parmi les plus avancés.
</Callout>

<Callout type="warning" title="Le bémol">
Facturation complexe et imprévisible. Interface dense et peu lisible. Données soumises au Cloud Act américain. Lock-in technique fort. Le support gratuit est limité. La courbe d'apprentissage est raide pour les débutants.
</Callout>

## Microsoft Azure : l'intégration Microsoft

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="Microsoft Azure, cloud intégré à l'environnement Microsoft" />

<AffiliateLink href="https://azure.microsoft.com/fr-fr/">Microsoft Azure</AffiliateLink> est le deuxième acteur du cloud mondial. Son atout principal : l'intégration native avec Windows Server, Active Directory, Microsoft 365 et les outils .NET.

Si votre entreprise tourne déjà sur des technologies Microsoft, Azure est le choix logique. La migration de serveurs Windows on-premise vers Azure se fait avec des outils dédiés (Azure Migrate).

Azure offre 200 dollars de crédits gratuits pendant 30 jours, plus des services toujours gratuits (750 heures de VM B1S, 5 Go de stockage Blob, base de données SQL).

Le réseau de datacenters couvre 60 régions, dont la France (France Centre à Paris, France Sud à Marseille). Azure est certifié HDS (Hébergement de Données de Santé) en France.

<Callout type="success" title="Ce qui le rend unique">
Intégration native avec tout l'environnement Microsoft (Windows Server, AD, M365, .NET). 60 régions dont deux en France. Certifié HDS pour les données de santé. 200 dollars de crédits gratuits. Services hybrides (Azure Arc) pour gérer le cloud et le on-premise ensemble.
</Callout>

<Callout type="warning" title="Le bémol">
La console est lourde et lente. La tarification est aussi complexe qu'AWS. Le lock-in Microsoft s'ajoute au lock-in cloud. Les prix sont élevés par rapport aux alternatives européennes. Le support en français n'est pas toujours disponible.
</Callout>

## Hetzner : le meilleur rapport qualité-prix en Europe

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="Hetzner, hébergeur cloud allemand au meilleur rapport qualité-prix" />

<AffiliateLink href="https://www.hetzner.com/">Hetzner</AffiliateLink> est un hébergeur allemand fondé en 1997. Pas de marketing tape-à-l'oeil, pas de console surchargée. Juste des serveurs fiables à des prix qui défient la concurrence.

Un serveur cloud (CX22) avec 2 vCPU, 4 Go de RAM et 40 Go de SSD coûte 3,99 euros par mois. Chez AWS, une instance comparable coûte trois à quatre fois plus. Les serveurs dédiés commencent à 39 euros par mois.

Hetzner propose aussi du stockage objet compatible S3, des load balancers, du DNS et des volumes de stockage supplémentaires. Le tout avec une API propre et une CLI officielle.

Les datacenters sont en Allemagne (Falkenstein, Nuremberg) et en Finlande (Helsinki). Depuis 2022, Hetzner est aussi présent aux États-Unis (Ashburn, Hillsboro).

<Callout type="success" title="Ce qui le rend unique">
Prix imbattables en Europe. Datacenters en Allemagne et Finlande (RGPD natif). Interface simple et claire. API et CLI bien conçues. Serveurs dédiés abordables. Bande passante généreuse (20 To inclus). Facturation transparente à l'heure.
</Callout>

<Callout type="warning" title="Le bémol">
Catalogue de services limité (pas de fonctions serverless, pas de services managés avancés). Pas de région en France. Le support est en anglais et allemand uniquement. Pas adapté aux projets qui ont besoin de services cloud avancés (IA, big data, IoT).
</Callout>

## OVHcloud : le cloud souverain français

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="OVHcloud, hébergeur cloud français souverain" />

<AffiliateLink href="https://www.ovhcloud.com/fr/">OVHcloud</AffiliateLink> est le leader européen du cloud. Fondé à Roubaix en 1999, l'entreprise gère ses propres datacenters en France, en Allemagne, au Canada et à Singapour.

OVHcloud propose un catalogue complet : serveurs dédiés, instances cloud (Public Cloud basé sur OpenStack), Kubernetes managé, stockage objet, bases de données managées, et un service de Bare Metal as a Service.

Les prix sont compétitifs. Une instance cloud d2-2 (1 vCPU, 2 Go RAM, 25 Go SSD) démarre à 0,0070 euro de l'heure, soit environ 5 euros par mois. Les serveurs dédiés commencent à 59,99 euros par mois.

OVHcloud est certifié SecNumCloud par l'ANSSI. C'est le plus haut niveau de certification cloud en France. Pour les entreprises soumises à des contraintes réglementaires, c'est un argument de poids.

<Callout type="success" title="Ce qui le rend unique">
Datacenters en France. Certifié SecNumCloud (ANSSI). Basé sur OpenStack (pas de lock-in propriétaire). Prix compétitifs. Support en français. Bande passante illimitée sur les serveurs dédiés. Engagements RGPD forts.
</Callout>

<Callout type="warning" title="Le bémol">
La console a longtemps souffert de bugs et de lenteurs (ça s'améliore). L'incendie du datacenter de Strasbourg en 2021 a marqué les esprits. Le catalogue de services managés est moins fourni qu'AWS ou Azure. La documentation technique est inégale selon les produits.
</Callout>

## DigitalOcean : la simplicité avant tout

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="DigitalOcean, hébergeur cloud simple pour développeurs" />

<AffiliateLink href="https://www.digitalocean.com/">DigitalOcean</AffiliateLink> a construit sa réputation sur la simplicité. Là où AWS vous noie sous les options, DigitalOcean vous propose un serveur en trois clics.

Les Droplets (machines virtuelles) démarrent à 4 dollars par mois pour 1 vCPU, 512 Mo de RAM et 10 Go de SSD. Le plan le plus populaire (2 vCPU, 2 Go RAM, 60 Go SSD) coûte 18 dollars par mois.

DigitalOcean propose aussi du Kubernetes managé, des bases de données managées (PostgreSQL, MySQL, Redis, MongoDB), du stockage objet (Spaces) et une plateforme d'apps (App Platform) pour déployer du code sans gérer de serveur.

Les datacenters sont aux États-Unis, en Europe (Amsterdam, Francfort, Londres), au Canada, en Inde, en Australie et à Singapour.

La documentation est exceptionnelle. Les tutoriels communautaires couvrent à peu près tous les cas de figure. C'est la plateforme idéale pour apprendre le cloud.

<Callout type="success" title="Ce qui le rend unique">
Interface claire et simple. Déploiement en quelques clics. Documentation et tutoriels parmi les meilleurs du marché. Prix prévisibles et transparents. App Platform pour déployer sans gérer de serveur. Kubernetes managé abordable.
</Callout>

<Callout type="warning" title="Le bémol">
Pas de datacenter en France. Données soumises au Cloud Act (entreprise américaine). Moins de services que les grands clouds. Pas de certification SecNumCloud ou HDS. Les performances réseau sont correctes sans plus.
</Callout>

## Scaleway : le cloud européen pour développeurs

<CloudinaryImage src="blog-screenshots/google-cloud-home" alt="Scaleway, cloud européen pour développeurs" />

<AffiliateLink href="https://www.scaleway.com/fr/">Scaleway</AffiliateLink> est un hébergeur cloud français, filiale du groupe Iliad (Free). Les datacenters sont à Paris et Amsterdam.

Scaleway propose des instances cloud, du Kubernetes managé (Kapsule et Kosmos), du serverless (Functions et Containers), des bases de données managées, du stockage objet, et des GPU pour le machine learning.

Les prix sont compétitifs. Une instance DEV1-S (2 vCPU, 2 Go RAM, 20 Go SSD) coûte environ 0,01 euro de l'heure. Le stockage objet est facturé 0,015 euro par Go par mois.

L'interface est moderne. L'API est bien documentée. Le CLI (scw) est agréable à utiliser. C'est le cloud qui se rapproche le plus de l'expérience DigitalOcean, mais avec des datacenters en France et une conformité RGPD native.

<Callout type="success" title="Ce qui le rend unique">
Datacenters en France (Paris) et aux Pays-Bas. Conformité RGPD. Prix compétitifs. Interface moderne. Serverless (Functions et Containers). GPU pour le machine learning. CLI et API bien conçus. Facturation à la seconde.
</Callout>

<Callout type="warning" title="Le bémol">
Catalogue de services plus limité qu'AWS ou Azure. La communauté est plus petite. Quelques interruptions de service occasionnelles. Pas de certification SecNumCloud (contrairement à OVHcloud). Support en anglais principalement.
</Callout>

## Questions fréquentes

### Google Cloud Platform est-il gratuit ?

GCP propose un Free Tier avec 300 dollars de crédits pendant 90 jours. Après cette période, certains services restent gratuits dans des limites strictes (une instance e2-micro, 5 Go de stockage, 1 Go de bande passante). Pour un usage professionnel, les coûts montent vite.

### Quelle est l'alternative la moins chère à Google Cloud ?

Hetzner. Un serveur cloud avec 2 vCPU et 4 Go de RAM coûte 3,99 euros par mois. C'est trois à quatre fois moins cher qu'une instance comparable chez GCP, AWS ou Azure. Pour des besoins simples (hébergement web, API, base de données), c'est le meilleur rapport qualité-prix.

### Quel cloud choisir pour rester conforme au RGPD ?

OVHcloud (certifié SecNumCloud, datacenters en France), Hetzner (datacenters en Allemagne et Finlande) et Scaleway (datacenters à Paris). Ces trois hébergeurs sont européens et ne sont pas soumis au Cloud Act américain.

### AWS ou Azure : lequel choisir ?

AWS a le catalogue le plus large et la plus grande communauté. Azure est préférable si votre entreprise utilise déjà Windows Server, Active Directory ou Microsoft 365. En termes de prix, les deux se valent. La vraie différence se joue sur les compétences de votre équipe technique.

### Peut-on migrer facilement depuis Google Cloud ?

Ça dépend de votre architecture. Si vous utilisez des services propriétaires GCP (BigQuery, Cloud Functions, Firestore), la migration demande du travail. Si vous êtes sur des VMs ou du Kubernetes standard, la migration vers un autre cloud est plus simple. OVHcloud et Scaleway basés sur OpenStack facilitent les migrations depuis des environnements standards.

### DigitalOcean est-il adapté aux entreprises ?

Pour les startups et les PME, oui. La simplicité de la plateforme fait gagner du temps. Mais pour les grandes entreprises avec des besoins en conformité, en multi-régions ou en services managés avancés, AWS, Azure ou OVHcloud sont plus adaptés.
